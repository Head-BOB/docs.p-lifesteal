Extra Heart

Configuration for the Extra Heart item.

Template

heartItem:
  enabled: true
  name: "&c&lExtra Heart"
  material: "RED_DYE"
  lore:
    - "&6Gives an extra heart!"
    - "&7&oYou have ${chance}% to get an additional heart added."
  loseChance: 50
  addLimit: 0
  withdraw-enabled: true
  drop:
    mode: "never"

enabled

Defines whether the Extra Heart item is enabled. If set to false, the item cannot be used.

Type Boolean
Default true

name

The display name of the Extra Heart item. Supports color codes with & and hex colors with &#rrggbb.

Type String
Default &c&lExtra Heart

material

The material of the Extra Heart item. Must be a valid material from the Spigot documentation.

Type Material
Default RED_DYE

lore

The lore (description) shown on the Extra Heart item. Supports color codes and the ${chance} placeholder.

Type String List
Default ["&6Gives an extra heart!", "&7&oYou have ${chance}% to get an additional heart added."]

loseChance

The chance (in %) that the player will lose a heart instead of gaining one when using the Extra Heart item and the item fails. Set to 0 to disable heart loss on failure.

Type Integer (0-100)
Default 50

addLimit

The maximum amount of health points a player can reach by using the Extra Heart item. Set to 0 to disable the limit.

Type Integer
Default 0

withdraw-enabled

Whether the /lifesteal withdraw command is enabled, allowing players to convert one of their hearts into an Extra Heart item.

Type Boolean
Default true

drop.mode

Controls when hearts are dropped as items instead of being directly added to the killer's health bar.

Type String
Default never
Options never · always · limit_exceeded
  • never — Hearts are always added directly to the killer's health bar.
  • always — Hearts always drop as an item on the ground.
  • limit_exceeded — Hearts drop as an item only when the killer has reached the heart limit.
